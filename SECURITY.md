# Security Policy

If you find a security issue, do not open a public GitHub issue.

## Preferred reporting path

1. Use GitHub private vulnerability reporting if it is enabled for the repository.
2. If that is not available, contact the maintainer through a private channel.
3. Include impact, reproduction steps, affected browser version, and any proof-of-concept details needed to validate the report.

## Scope

Examples of issues worth reporting privately:

- Anything that could expose user data
- Anything that could trigger account actions
- Anything that could expand the extension beyond read-only behavior
- Supply-chain or dependency risks introduced into the repo

We will review reports, validate impact, and work toward a fix before public disclosure.
